# Graduation Project Final Documentation Requirements

## B.1 — General Formatting Guidelines

* **Document Length**: 50–70 pages (excluding cover, tables of contents, references, and appendices).
* **Font**: Times New Roman or Arial, 12pt for body text.
* **Line Spacing**: 1.5 for body text; single spacing for tables and captions.
* **Margins**: 2.5 cm on all sides.
* **Page Numbers**: Bottom center, starting from Chapter 1.
* **Figures and Tables**: High resolution; numbered and captioned; listed in a separate List of Figures/Tables.
* **References**: IEEE format; minimum 15 references (note: section B.1 lists minimum 10, but section B.2 references lists minimum 15; we will follow the stricter rule of 15).
* **Language**: English (unless the department head approves Arabic; bilingual abstract required in all cases).
* **Submission Format**: PDF via the faculty platform, plus ONE bound printed copy.
* **Submission Deadline**: As announced in Appendix B.4 (Delivery Schedule).

---

## B.2 — Required Document Structure

All graduation project documents must follow the structure below. Departments may add domain-specific sub-sections within the defined chapters without removing any required element.

### Cover Page
* University name, faculty name, department name
* Project title
* Academic year
* 'Graduation Project' designation and document type (Phase 1 / Final)
* Supervisor(s) name(s) and title(s)
* Team member names and student IDs

### Front Matter
* Table of Contents (with page numbers)
* List of Figures
* List of Tables
* List of Abbreviations
* Abstract (Arabic and English, 150–250 words each): problem, approach, outcome

### Chapter 1: Introduction
* **1.1 Background and Motivation**
* **1.2 Problem Definition**
* **1.3 Project Objectives and Proposed Solution**
* **1.4 Project Scope and Limitations**
* **1.5 Development Methodology**
* **1.6 Tools and Technologies Used (Software and Hardware)**
* **1.7 Project Timeline / Gantt Chart**
* **1.8 Report Organization**

### Chapter 2: Related Work
* Overview of existing systems, research, or solutions in the same domain.
* Comparative analysis: what distinguishes this project from existing work.
* All sources properly cited using IEEE format.

### Chapter 3: System Analysis
* **3.1 Stakeholder Analysis**
* **3.2 Functional Requirements**
* **3.3 Non-Functional Requirements**
* **3.4 Use Case Diagrams with actor descriptions**
* **3.5 Feasibility Study (technical, economic, and operational — brief)**

### Chapter 4: System Design
* **4.1 System Architecture / Component Diagram**
* **4.2 Class Diagrams**
* **4.3 Sequence Diagrams (for key use cases)**
* **4.4 Database Entity-Relationship Diagram (ERD) — if applicable**
* **4.5 User Interface Wireframes / Mockups**
* **4.6 Security Design Considerations**
* *Note: Research-track projects replace design chapters with Research Methodology, Experimental Setup, and Data Analysis chapters.*

### Chapter 5: Implementation
* **5.1 Implementation Environment and Setup**
* **5.2 Key Implementation Details (algorithms, modules, major decisions)**
* **5.3 Sample Screenshots / Output Samples**

### Chapter 6: Testing and Evaluation
* **6.1 Testing Strategy**
* **6.2 Test Cases and Results (table format)**
* **6.3 Performance Evaluation / Benchmarking (where applicable)**
* **6.4 Limitations and Known Issues**

### Chapter 7: Conclusion and Future Work
* **7.1 Summary of Achievements**
* **7.2 Lessons Learned**
* **7.3 Future Enhancements**

### References
* IEEE-format numbered reference list.
* Minimum 15 references; all must be cited in the text.

### Appendices (optional)
* User manuals, source code excerpts, raw data tables, additional diagrams.
