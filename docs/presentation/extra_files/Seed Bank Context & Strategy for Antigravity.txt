4. The Core Technology: Hierarchical Two-Stage Pipeline
Instead of one massive model, the system decouples tasks to handle visual noise and variance effectively.
* Stage 1: Object Localization (Inter-Class Variance)
   * Goal: Find the seed and isolate it from the background.
   * Primary Model: Faster R-CNN (Utilizes FPN for multi-scale detection and CBAM for attention). Performance: mAP@50 = 0.9835. Inference time = ~0.11s.
   * Alternative Real-Time Model: YOLOv8-Small. Used for conveyor belts. Performance: mAP@50 = 0.941. Inference time = 5.5ms.
* Stage 2: Defect Classification (Intra-Class Variance)
   * Goal: Determine if the isolated seed is healthy or defective.
   * Base Model: ResNet-18 (Chosen for being lightweight).
   * Custom Modifications (Crucial AI aspect):
      1. Stride Reduction to (1,1): Prevents over-downsampling to see tiny defects.
      2. CBAM (Attention Mechanism): Forces the model to look at discoloration/irregularities, not just background.
      3. Hybrid Pooling Strategy: Combines Max Pooling and Average Pooling (GMP + GAP) to capture both general patterns and sharp anomalies (like black lines).
* Data & Performance metrics:
   * Evaluated 12 datasets. Used datasets for Maize and Coffee.
   * Maize Results (Cleaned Data): Accuracy 83.18%, F1-Score 0.7687, Recall 0.8894.
   * Coffee Results (V3 Hybrid Model): F1-Score 0.9101, Recall 0.9335.
5. System Architecture & Engineering (The IS Aspect)
* Frontend (Vanilla JavaScript + Tailwind CSS):
   * Why? Zero build process (rapid prototyping), minimal bundle size (fast loading for field use), direct DOM manipulation is sufficient for UI needs, easy API integration.
* Backend (Python + FastAPI):
   * Why Python? Dominates ML/AI ecosystem (PyTorch, TensorFlow, OpenCV, albumentations), rapid prototyping, seamless GPU/CUDA support.
   * API Layer: Central hub for image uploads, guest/auth handling, and asynchronous processing.
* Database (PostgreSQL):
   * Why? Native ACID compliance for relational data, JSONB support for flexible metadata, excellent Python (SQLAlchemy) integration.
   * Schema Hierarchy: User -> ScanBatch -> ScanImage -> SeedDetection.
* Architecture Pattern (Monolithic):
   * Why? Single codebase for fast dev cycles, lower cost, easy debugging, and tight ML pipeline integration. Fits the small team size perfectly.
* Unified User Model: Uses device fingerprinting to allow Guest access without traditional logins.
